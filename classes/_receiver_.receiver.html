Class Receiver

PSK2 Receiver class that listens for and accepts incoming payments.

The same Receiver may be used for accepting single-chunk payments, streaming payments, and chunked payments.

It is recommended to use the createReceiver function to instantiate Receivers.

generateAddressAndSecret

  • generateAddressAndSecret(keyId?: Buffer): object
    • Defined in receiver.ts:226

    Generate a unique ILP address and shared secret to give to a sender.

    The Receiver must be connected before this method can be called.

    Note: A single shared secret MUST NOT be given to more than one sender.

    Parameters

    • Optional keyId: Buffer

      Additional segment that will be appended to the destinationAccount and can be used to correlate payments. This is authenticated but unencrypted so the entire Interledger will be able to see this value.

registerRequestHandlerForSecret

  • registerRequestHandlerForSecret(sharedSecret: Buffer, handler: RequestHandler): object
    • Defined in receiver.ts:246

    Register a RequestHandler for a specific sharedSecret. This will be called instead of the normal requestHandler when requests come in for the destinationAccount returned by this function.

    This is especially useful for bidirectional protocols built on top of PSK2 in which both sides want to use the same sharedSecret.

    Parameters

    • sharedSecret: Buffer

      Secret to use for decrypting data and generating fulfillments

    • handler: RequestHandler

      Callback that will be called when a request comes in for the destinationAccount returned by this function

    Returns object

    • destinationAccount: string
    • sharedSecret: Buffer
